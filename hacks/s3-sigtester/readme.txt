***Amazon S3 SIGNATURE TESTER***

This tool is designed to help developers validate their code for signing requests to Amazon S3.
Common problems include constructing the "string-to-sign" incorrectly, using code that generates a signature incorrectly, and not properly URL encoding signatures (for query strings only).
Please see the tool itself for more detailed guidance in identifying signing problems.

Prerequisites: A web browser. This tool has been tested on Firefox 1.0.7 and Internet Explorer 6.0. If you have difficulty using the tool, please try one of these browsers.


Amazon has always told its AWS customers to never share their AWS Secret Access Key with anyone, especially never type it into an html text field. In order to facilitate analysis of your application's generation of signatures, the examples below are provided.
Using the examples: Each example contains the HMAC-Sha1 signature of the "string to be signed" using "fake-secret-key" as the key. Your code should generate the same signature for each string-key combination. Failure to do so incicates a problem with your signing code.
The hex encoded string is the hex encoded request returned by Amazon when a authentication fails. To make sure your code is constructing the "string to be signed" correctly, sign a request to Amazon S3 using any fake secret key. Copy and paste the returned hex encoded string that is returned 
in the error document into the signature tester. Type in your fake secret key and click "Get Signature." Compare the "sting to be signed" that the tool produces with the one your code produces. These should match. Failure here indicates that your code is not correctly generating the "string
to be signed." To make sure your code is correctly generating URL encoded query strings, follow the same process and compare the URL encoded signature with the one generated by your code. If these do not match, there is a problem with the way your code URL encodes the signature.
